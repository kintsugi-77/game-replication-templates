# 玩法实施卡 · 模板与交活 Prompt（通用）

> **用途**：游戏复刻实现阶段,每个里程碑填一张「玩法实施卡」,连同「交活 Prompt」发给执行 AI。
> **主流程、纪律、建工程、推 dev、验收全在 runbook**;这份只管「把原作拆成里程碑、每段做哪块玩法逻辑」。真值数字统一放 `docs/<game>-spec.md`,卡**只引用、不抄数**。
> ★ 通用节奏:**先做最小可玩闭环(及格线),再逐系统铺开(评优线,把"最区别于同类的灵魂系统"重点做),最后打磨。**

## 模板 A · 玩法实施卡(两段式)

**导演填这 5 栏**(都拍得了板,不碰代码):
```
① 目标：这次能玩到核心循环的哪一段(玩家视角一句话)
② 及格判定：真窗口里"什么状态变了"算成功(可观察/可量化)
③ 真值/CONFIG：这次要对的数指向 spec 哪节;没定的标"占位/待拍板"
④ 验收点：对标原作我会盯哪几处
⑤ 不做/边界：这次切片不碰什么(防越界)
```
**执行 AI 在"动手前先做"回填、导演审这 5 栏**(导演填不了也不该填):
```
behavior-first 判断(哪些只改声明 / 哪些写逻辑)
要用的 object·behavior(+ declaration 出处)
要用的运行时 API(+ types/*.d.ts 出处)
参考哪个 example
状态机 phase 设计
```

## 模板 B · 交活 Prompt(把卡贴进【这次要做的】)
```
按 runbook 的角色与纪律做这个切片(声明/逻辑分离、behavior-first、先查 .d.ts、真窗口验证)。真值见 docs/<game>-spec.md,CONFIG 只引用不抄数。

【这次要做的】
<把下面对应里程碑的实施卡贴在这里>

【动手前先做(等我确认再写代码)】
1. 复述你对这个切片核心循环的理解。
2. 回填技术 5 栏：behavior-first / object·behavior / API / 参考范例 / 状态机 phase,各标出处。
我确认后你再写;写完先离线验证,再推 dev + 真窗口自测。
```

## 里程碑拆分指引(替代具体游戏的 M1…Mn)
- **M1 = 最小可玩闭环(及格线)**：核心循环跑通一轮,数值可先占位。
- **M2 … Mk = 逐系统铺开(评优线)**：每次接一个系统的真值并对拍原作;把**灵魂系统**(最区别于同类的)排在中段重点做。
- **M_last = 经营/体验打磨**：UI/HUD、存档、音效、美术替换占位。
- 每个里程碑用模板 A 填一张卡。

### 空白里程碑卡(复制即填)

## 附：两张小抄(帮你审执行 AI 回填的技术栏)
**scene.logic 固定段式**：类型别名 → register → CONFIG → 主入口 → createState → handlers。

**behavior-first 怎么判**：
- 能力/表现类(行走、相机跟随、拾取、碰撞、物理)→ 先试纯 behavior 配置(只改声明)。
- 游戏规则类(状态机、经济、数值系统)→ 写逻辑。
- 拿不准 → 让执行 AI 在"动手前先做"说明判断和出处,你再放行。
